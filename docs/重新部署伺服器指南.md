# 重新部署伺服器到 Railway

## 🚀 快速重新部署步驟

由於你修改了伺服器端程式碼（CORS 配置），需要重新部署到 Railway：

### 方法 1：自動部署（推薦）

如果你已經連接到 GitHub：

1. **提交並推送程式碼**
   ```bash
   git add .
   git commit -m "修復 CORS 配置，允許 localhost"
   git push
   ```

2. **Railway 會自動部署**
   - Railway 偵測到程式碼變更
   - 自動執行 `npm run build`
   - 自動執行 `npm run start`
   - 等待 3-5 分鐘

3. **查看部署狀態**
   - 前往 https://railway.app
   - 點擊你的專案
   - 查看 "Deployments" 標籤
   - 等待狀態變成 "Success"

### 方法 2：手動觸發部署

如果自動部署沒有觸發：

1. **前往 Railway Dashboard**
   - https://railway.app
   - 登入你的帳號

2. **找到你的專案**
   - 點擊 "Wevro" 專案

3. **手動觸發部署**
   - 點擊服務（Service）
   - 點擊 "Deployments" 標籤
   - 點擊 "Redeploy" 按鈕
   - 選擇最新的提交

4. **等待部署完成**
   - 查看構建日誌
   - 確認沒有錯誤
   - 等待狀態變成 "Success"

## ✅ 驗證部署成功

部署完成後：

1. **測試 API 端點**
   ```bash
   # 測試健康檢查（如果有點）
   curl https://www.wevro.co/api/auth/user
   ```

2. **在模擬器中測試**
   - 開啟 Android 模擬器
   - 執行應用
   - 測試生成例句功能
   - 測試生成同義字功能
   - 不應該再出現 CORS 錯誤

## 🔍 查看部署日誌

如果部署失敗：

1. **查看構建日誌**
   - Railway Dashboard → Deployments
   - 點擊失敗的部署
   - 查看 "Build Logs"

2. **查看執行日誌**
   - Railway Dashboard → Service
   - 點擊 "Deployments"
   - 點擊部署 → "View Logs"

## 📝 注意事項

- ⚠️ 部署期間服務會短暫中斷（幾秒鐘）
- ✅ 部署完成後自動恢復
- 📊 可以透過 Railway Dashboard 監控服務狀態
